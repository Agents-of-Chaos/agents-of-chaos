### CONCERN flagged for controller review

The brief's expected result was `exit=1` (no matches). The actual result is `exit=0` because the overlay-handling client code (`warm_path` property accessors, "warm paths only" UI string) ships unconditionally in the JS bundles for both `/funding` and `/networks`. The architecture loads the overlay DATA only at Astro build time (server-side, `import.meta.env.DEV` guard in the Astro frontmatter); the overlay-reading CLIENT code ships regardless.

This is a structural decision from Tasks 1–7: the JS islands always include overlay-reading logic; the data path is protected by an Astro-level guard. The DATA invariant holds; the CODE-ships-without-data design is intentional.

If the brief intended `exit=1`, that would require wrapping the entire overlay-reading client path behind a dead-code-eliminated DEV flag (e.g. `if (import.meta.env.DEV) { ... }` so the bundler tree-shakes it in prod). That is a Task 1–7 fix, not Task 8.
